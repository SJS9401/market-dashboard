### 파일 2: feedback_quarterly_review_system.md

```markdown
---
name: quarterly-review 3-stage 워크플로우 전체 그림
description: quarterly-review 시스템의 Stage 1~3 흐름과 earnings 3종 모드와의 자동 연계 조건 (cron 스케줄·섹터 매칭·누락 안내).
type: feedback
---

quarterly-review 시스템은 3-stage 워크플로우로 구성되며, earnings 3종 모드 산출물을 자동으로 로드해 섹터·분기 단위 통합 분석을 수행한다.

**Why:** 분기 실적 시즌이 끝날 때마다 종목별 분석을 수동으로 모아서 섹터 단위로 재가공하는 작업이 반복적이었다. earnings deep-dive 산출물을 표준 위치/메타데이터로 저장하면, quarterly-review가 자동으로 섹터별 종목을 모아 분기 주도 섹터를 판정할 수 있다.

**How to apply:** earnings 3종 모드 산출물은 항상 quarterly-review가 검색 가능한 형태로 저장한다. SKILL.md에 아래 3-stage 흐름과 자동화 조건을 반드시 안내한다.

(1) Stage 1 — [실적 심층분석 모드]
→ 기업 단위 분석
→ 출력: `WORK_DIR/earnings-deep-dive/{YYYY-Q}_{종목}_심층.md`

(2) Stage 2 — [분기 섹터 분석 모드]
→ 섹터 단위 통합 분석
→ Stage 1의 deep-dive .md 파일을 자동 로드
→ 메타데이터 헤더의 [섹터] 필드로 워치리스트 [분기 주도 섹터]와 매칭
→ 누락 종목이 있으면 사용자에게 "[종목] 심층분석 .md 미존재" 안내

(3) Stage 3 — quarterly-review 자동 스케줄러
→ cron: `0 8 16,31 3,5,8,11 *`
→ 본격 실행 일자: 5/16, 8/16, 11/16, 3/31 오전 8시 KST
→ Stage 2 결과를 분기 단위로 누적 비교

**자동화 작동 조건 (3개 모두 충족 필요):**
(1-1) 모든 deep-dive .md가 표준 위치에 저장되어 있어야 함 → `WORK_DIR/earnings-deep-dive/{YYYY-Q}_{종목}_심층.md`
(1-2) 메타데이터 헤더의 [섹터] 필드가 워치리스트 [분기 주도 섹터]의 명칭과 정확히 일치해야 함
(1-3) 분기 식별자(YYYY-Q)가 일관성 있게 사용되어야 함 (예: 2026-Q1)

**누락 시 동작:**
→ Stage 2가 자동 로드 시도 → 누락 종목 발견 → "[종목] 심층분석 .md 미존재" 메시지로 사용자에게 안내 → 사용자가 해당 종목 [실적 심층분석 모드] 수동 실행 후 Stage 2 재시도
```

### 파일 3: feedback_daily_issue_hierarchy.md

```markdown
---
name: 본문 위계 5단계 표기 룰 (일일 이슈 + 실적 분석 공용)
description: 분석 본문 작성 시 ## / ① / (1) / (1-1) / → 5단계 위계를 일관되게 사용하는 룰. 실적 3종 모드와 일일 이슈 분석에 모두 적용.
type: feedback
---

분석 본문은 항상 5단계 위계로 작성한다. 단계 마커를 임의로 변경하지 않는다.

**Why:** 위계 마커가 들쑥날쑥하면 같은 분기 내 종목·섹터 간 흐름 비교가 어렵다. quarterly-review가 deep-dive .md를 자동 파싱해 섹터 단위로 통합할 때도 위계가 일관되어야 정확하게 추출된다. 또한 5단계 이상은 가독성을 떨어뜨려 의도적으로 4단계 + 화살표(→)로 잎 노드를 표기한다.

**How to apply:** 본문 작성 시 다음 5단계 마커를 그대로 사용한다.

(1) 1단계 — 대주제: `## 헤더`
→ 예: `## 실적 리뷰 결과`

(2) 2단계 — 중주제: `①` `②` `③` (원형 숫자)
→ 예: `① 매출/영업이익 결과`

(3) 3단계 — 소주제: `(1)` `(2)` `(3)` (괄호 숫자)
→ 예: `(1) 컨센서스 대비`

(4) 4단계 — 세분화: `(1-1)` `(1-2)` (하이픈 연결)
→ 예: `(1-1) 매출 +5% beat`

(5) 5단계 — 내용: `→` (화살표로 시작)
→ 예: `→ 매출 19.1조원 +5% YoY`

**금지 사항:**
→ 5단계 이상 위계 만들지 말 것 (예: (1-1-1) 금지)
→ ① 대신 1., A., 가. 등 다른 마커 사용 금지
→ → 대신 -, *, • 사용 금지 (잎 노드는 항상 →)

**적용 범위:** 실적 프리뷰/리뷰/심층분석 .md 본문, 일일 이슈 분석, 분기 섹터 분석 등 모든 .md 산출물에 동일 적용.
```

### 파일 4: feedback_fng_korean_label.md

```markdown
---
name: F&G(Fear & Greed) 한글 공식 영역명 표기 룰
description: CNN Fear & Greed Index를 분석에서 인용할 때 영역명은 한글 공식 5단계만 사용 (극단적 공포·공포·중립·탐욕·극단적 탐욕).
type: feedback
---

CNN Fear & Greed Index의 점수 영역을 표기할 때는 한글 공식 5단계만 사용한다. 의역·영문 혼용·자체 변형 명칭은 금지한다.

**Why:** 같은 점수대를 다양한 명칭으로 부르면(예: "광기" vs "극단적 탐욕") 섹터·분기 비교 시 표기 불일치로 검색·집계가 어려워진다. CNN의 공식 5단계 분류를 그대로 한글화한 명칭을 표준으로 사용한다.

**How to apply:** F&G 점수 인용 시 다음 5개 영역명만 사용한다.

(1) 0~24: **극단적 공포**
(2) 25~44: **공포**
(3) 45~55: **중립**
(4) 56~74: **탐욕**
(5) 75~100: **극단적 탐욕**

**예시:**
→ OK "현재 F&G 지수 18 (극단적 공포)"
→ OK "F&G 62 (탐욕) 구간 진입"
→ NG "F&G 18 (Extreme Fear)" — 영문 사용 금지
→ NG "F&G 78 (광기/환희)" — 의역 금지
→ NG "F&G 50 (Neutral)" — 영문/한글 혼용 금지
→ NG "F&G 30 (공황)" — 자체 변형 명칭 금지

**점수 인용 형식:**
→ 항상 점수 + 영역명 병기: "F&G {점수} ({영역명})"
→ 영역 경계값(24/25, 44/45, 55/56, 74/75)은 더 보수적인 영역으로 분류 (예: 25 → 공포, 75 → 극단적 탐욕)

**적용 범위:** 실적 분석 본문에서 시장 심리 인용 시, 일일 이슈 분석, 전략 분석 등 F&G가 등장하는 모든 .md 산출물.
```
